# テストルール

## ファイル配置

- テストファイルはソースと同じディレクトリに `*.test.ts` で配置する
- テストファイル名はソースファイル名に `.test` を付ける（例: `service.ts` → `service.test.ts`）

## テストコードの書き方

- `describe` でテスト対象（関数名・クラス名）をグループ化する
- `it` の説明は日本語で、テストの意図が分かるように書く
- テストデータのファクトリ関数はテストファイル内に定義する
- `vi.mock` はファイルトップレベルに配置する（import より前）

## モック

- 外部依存は `vi.mock` でモックする
- Repository 層のモックは Service テストでのみ行う

## やらないこと

- スナップショットテストは使わない（変更に弱い）
- カバレッジの数値目標は設けない（意味のあるテストを優先する）
