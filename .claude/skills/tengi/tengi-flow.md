# 天議フロー詳細

## 作業ディレクトリ

引数で渡された天議ディレクトリを作成し、以下のファイルで情報を共有する：

| ファイル              | 内容                                 |
| --------------------- | ------------------------------------ |
| `topic.md`            | 議題・コンテキスト・方針候補（任意） |
| `counter-example.md`  | 反例スキルの出力                     |
| `step1-opinions.md`   | 意見出しフェーズの集計結果           |
| `step2-discussion.md` | 議論フェーズの集計結果               |

## topic.md のフォーマット

```markdown
## 議題
〇〇

## コンテキスト
〇〇

## 方針候補（任意）

### 方針A: 名前
説明

### 方針B: 名前
説明
```

方針候補は任意。排他的な選択肢がある場合に記載する。なければ省略してよい

## フロー

### ステップ1: インプット解析

引数の議題を元に `tengi/topic.md` を作成する
方針候補は排他的な選択肢がある場合のみ記載する。自由評価型の議題では省略する

### ステップ2: 反例スキル起動（先行）

**`/adversary {tengi_dir}`** を起動し、結果を `tengi/counter-example.md` に書き出す

| 種別     | 重大度 | 動作                                                 |
| -------- | ------ | ---------------------------------------------------- |
| 議題の穴 | 致命的 | 致命的問題を人間に報告して終了（メンバー起動しない） |
| 議題の穴 | 軽微   | ステップ3へ進む                                      |
| 方針の穴 | -      | ステップ3へ進む                                      |

### ステップ3: 意見出し

Agent tool × N（天議メンバー、N = パラメータで指定された人数）を並列起動する

- 各メンバーにメンバー番号と割り当てた観点を明示的に伝える
- `tengi-member.md` のルールを渡す
- 天議ディレクトリのパスを渡す（`counter-example.md` も読める状態）

#### 3人の場合

- メンバー1〜3: 議題に応じた異なる観点をそれぞれ割り当てる（例：実装コスト、保守性、ユーザ体験など）

#### 5人の場合

- メンバー1〜3: 議題に応じた異なる観点をそれぞれ割り当てる（例：実装コスト、保守性、ユーザ体験など）
- メンバー4〜5: メンバー1〜3に割り当てた観点のリストを伝え、それ以外の観点を自分で考えて決めるよう指示する

### ステップ4: 意見出し集計

全メンバーの意見を集計し `tengi/step1-opinions.md` に書き出す。集計時はメンバーの原文からキーフレーズを保持し、過度に要約しないこと：

```markdown
## ステップ1 集計

| メンバー | 観点       | 立場の要約（1行） |
| -------- | ---------- | ----------------- |
| 1        | 実装コスト | 〇〇              |
| 2        | 保守性     | 〇〇              |
| ...      | ...        | ...               |

### メンバー1
- 観点: 実装コスト
- 分析: 〇〇
- 提案: 〇〇
- 懸念: 〇〇

...

## 反例スキルの指摘
- 〇〇

## 反例スキルの対案（ある場合のみこの項目を書く）
- 〇〇
```

### ステップ5: 議論

議論モードによって動作が異なる

#### agent モード

新しい Agent tool × N を並列起動する（ステップ3と同じ人数）

##### 渡す情報
- 各メンバーにステップ1のメンバー番号・観点・意見を明示的に伝える（step1-opinions.md の該当メンバーの意見を引用）
- `step1-opinions.md` の**全文**を読ませる
- `counter-example.md` も読ませる

##### 行動指示
- 「全員の意見を踏まえた上で、自分の立場を再評価せよ。納得した論拠があれば立場を変えてよい。反論があれば具体的に応答せよ」と指示する

#### solo モード

進行役が `step1-opinions.md` と `counter-example.md` を読み、以下を行う：

- メンバー間で矛盾・対立している意見を特定する
- 各提案に対する賛成・反論の有無を整理する
- 反例スキルの指摘と各メンバーの意見の関係を整理する

結果を `tengi/step2-discussion.md` に書き出す：

```markdown
## 進行役による突き合わせ

### 提案の整理
- 提案X: メンバーA・Bが支持、反論なし
- 提案Y: メンバーCが提案、メンバーDが懸念を表明（内容）
- ...

### 対立点
- 〇〇についてメンバーAとメンバーBの意見が対立（それぞれの論拠を併記）

### 反例スキルとの関連
- 反例スキルの指摘Xに対して、メンバーAが対応策を提案済み
- ...
```

solo モードではステップ6の集計は不要（上記がステップ6を兼ねる）。ステップ6のカテゴリ分類へ直接進む

### ステップ6: 議論集計・最終集約

#### agent モードの場合

全メンバーの意見を集計し `tengi/step2-discussion.md` に書き出す。集計時はメンバーの原文からキーフレーズを保持し、過度に要約しないこと：

```markdown
## ステップ2 集計

| メンバー | 観点       | ステップ1からの変更 |
| -------- | ---------- | ------------------- |
| 1        | 実装コスト | なし / あり（要約） |
| ...      | ...        | ...                 |

### メンバー1
- 観点: 実装コスト
- ステップ1からの変更: なし / あり（内容）
- 分析: 〇〇
- 提案: 〇〇
- 懸念: 〇〇
- 他メンバーへの応答: 〇〇

...
```

#### カテゴリ分類（両モード共通）

進行役が `step1-opinions.md` と `step2-discussion.md` を参照し、以下のカテゴリに分類する

「言及」の定義: メンバーが当該提案に対して賛成・反対・条件付き賛成のいずれかの立場を明示的に表明したこと（単なる存在確認や分析中の示唆は含まない）。言及はステップ1・ステップ2の両方を通じて判定する

- **合意が高い提案**: 複数メンバーが提案し、反論が出ていない
- **少数だが反論なしの提案**: 1人の提案で、少なくとも1人以上が言及した上で反論が出ていない
- **未検討の提案**: 提案者以外から言及がなかった提案。ステップ2で初出の提案もここに分類する
- **意見が割れた提案**: 賛否が分かれている（対立する論拠を併記）

進行役は「何を採用するか」を判断しない。分類のみ行う

### ステップ7: 改善項目の整理

進行役がカテゴリ分類の結果を、天議ディレクトリの親ディレクトリに `improvements.md` としてチェックリストで書き出す。この時点では採用・見送りの判断はしない：

各項目にはカテゴリごとのIDを付与する（`a` = 合意が高い、`b` = 少数だが反論なし、`c` = 未検討、`d` = 意見が割れた）：

```markdown
# 改善項目

## 合意が高い
- [ ] a-1: 〇〇
- [ ] a-2: 〇〇

## 少数だが反論なし
- [ ] b-1: 〇〇

## 未検討
- [ ] c-1: 〇〇

## 意見が割れた
- [ ] d-1: 〇〇
```

人間がこのファイルを確認し、各項目の採用・見送りを判断する。判断後、反映は上から順に行い、完了したらチェックを入れる。依存関係がなければ順に、相互依存する項目はまとめて処理可

#### 項目の状態

- `- [ ] a-1: 〇〇`: 未判断
- `- [x] a-1: 〇〇`: 採用・対応済み
- `- 見送り: a-1: 〇〇`: 見送り。元の詳細は残し、見送り理由を追記する

```markdown
- 見送り: b-2: 〇〇
  - 見送り理由: 〇〇
  - （元の詳細はそのまま残す）
```

## アウトプット

ステップ7完了後、呼び出し元に以下を返す:

- 正常完了: 「improvements.md を作成しました」
- 致命的問題: 「致命的な問題が見つかりました。天議を停止します。（説明）」
