# セミナーダイジェスト動画生成アプリ

## 概要

セミナー動画（mp4）をアップロードすると、AIがパンチライン（最も印象的・重要な部分）を自動抽出し、約5分のダイジェスト動画を生成するWebアプリケーション。

## 機能要件

### MVP

- mp4 動画のアップロード（最大90分）
- 自動文字起こし
- AIによるパンチライン抽出
- ダイジェスト動画の生成（約5分）
- 生成された動画のダウンロード

### 将来対応（スコープ外）

- ユーザー認証（Firebase or Clerk）
- 抽出基準の選択（パンチライン / トピック網羅 / カスタム）
- 課金・利用制限

## 非機能要件

- 一般公開（認証なし）
- 90分の動画を処理できること
- Railway でホスティング可能であること

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Next.js (App Router) + TypeScript |
| バックエンド | Next.js API Routes / Server Actions |
| 文字起こし | OpenAI Whisper API |
| パンチライン抽出 | Claude API (claude-sonnet-4-20250514) |
| 動画編集 | FFmpeg (fluent-ffmpeg) |
| ストレージ | Railway Volume |
| ジョブキュー | Inngest |
| インフラ | Railway |

## アーキテクチャ

```
[ユーザー]
    │
    ▼
[Next.js Frontend]
    │ アップロード
    ▼
[API Route] ──────► [Railway Volume]
    │                     │
    │ ジョブ登録           │
    ▼                     │
[ジョブキュー]             │
    │                     │
    ▼                     ▼
[Worker Process] ◄─────────┘
    │
    ├─► [Whisper API] 文字起こし
    │
    ├─► [Claude API] パンチライン抽出
    │
    ├─► [FFmpeg] 動画切り出し・結合
    │
    └─► [ストレージ] ダイジェスト保存
           │
           ▼
      [ユーザーに通知 → ダウンロード]
```

## 処理フロー

1. **アップロード**: ユーザーが mp4 をアップロード
2. **保存**: ストレージに保存、ジョブをキューに登録
3. **文字起こし**: Whisper API で全文をテキスト化（タイムスタンプ付き）
4. **分析**: Claude API に文字起こしを渡し、パンチラインの時間範囲を特定
5. **切り出し**: FFmpeg で該当部分を切り出し
6. **結合**: 複数のクリップを結合して5分程度のダイジェストを生成
7. **通知**: 処理完了をユーザーに通知（ポーリング or WebSocket）
8. **ダウンロード**: ユーザーがダイジェスト動画をダウンロード

## Claude API プロンプト設計

### 入力

- 文字起こしテキスト（タイムスタンプ付き）
- 目標時間（例: 5分）

### 出力

JSON形式で、抽出すべきセグメントのリストを返す：

```json
{
  "segments": [
    {
      "start": "00:05:30",
      "end": "00:06:45",
      "reason": "このセミナーの核心となる主張",
      "quote": "「〇〇こそが成功の鍵なんです」"
    },
    ...
  ],
  "total_duration": "04:52"
}
```

### 抽出基準（プロンプトに含める）

- 話者が強調している箇所
- 聴衆の反応が想定される発言
- 具体的な事例・エピソード
- 結論・まとめの部分
- 印象的なフレーズ・名言

## UI/UX

### 画面構成

1. **トップページ**: アップロードエリア + 使い方説明
2. **処理中画面**: 進捗表示（文字起こし中 → 分析中 → 動画生成中）
3. **完了画面**: プレビュー + ダウンロードボタン

### インタラクション

- ドラッグ&ドロップでアップロード
- 処理中は進捗バー表示
- 完了後はブラウザ内でプレビュー再生可能

## 制約・リスク

### 技術的制約

- **Whisper API の制限**: 25MB/リクエスト → 長い動画は音声を分割して送信
- **Railway の制限**: リクエストタイムアウト、メモリ制限 → 非同期ジョブ処理で対応
- **FFmpeg**: Railway で動かすには Docker イメージにインストールが必要

### リスク

- 処理時間が長い（90分動画 → 数分〜十数分かかる可能性）
- APIコストが高くなる可能性（Whisper + Claude）
- 動画によっては「パンチライン」が不明確な場合がある
